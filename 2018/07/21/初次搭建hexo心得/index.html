初次搭建hexo心得

2018.07.21 xin tai  热度 ℃

总述

搭建的过程分为以下几步
1 安装node.js
2 安装git
3 安装hexo博客框架
4 测试本地
5 部署config以及github

安装node.js

Node.js下载地址：https://nodejs.org/en/download/ 安装过程默认

安装git

Git软件下载地址：https://git-scm.com/download
安装过程默认

安装hexo

Hexo官方网站：https://hexo.io/zh-cn/
1 在本地找一个地方用来存放hexo
2 npm install -g hexo-cli
3 hexo init #初始化
4 hexo install #安装依赖包

#测试本地运行
hexo g #等同于hexo generate 生成静态文件
hexo s #hexo server 启动本地服务

#部署到github
1 在github创建一个项目，比如我的是tai769 则项目名称为 tai769.github.io，github.io不变，前面的最好用github的名字，至于为什么我也不知道，因为本身也是菜鸟学习。

#部署到线上
1 打开_config.yml 找到ploy 修改成以下的 repo源填写你自己的项目地址
deploy:
type: git
repository: https://github.com/tai769/tai769.github.io.git
branch: master
2 设置ssh连接
cd ~/.ssh
ssh-keygen -t rsa -C “2217687104@qq.com“ # 这个地方添个人邮箱
将生成的pub文件里面的内容复制粘贴
3 登录github 设置 找到ssh和gpg keys
新建一个new ssh key 讲复制内容粘贴进去
4 输入 ssh -T git@github.com 测试是否连接成功。如果看到hi后面是你的用户名，就说明成功了。

使用

hexo clean
hexo generate
hexo deploy

心得

要注意ssh这个地方，要对应新的ssh-key 不然就会出现出错，导致你的本地可以访问，远程却不可以，至于为什么我也不知道，多看日志报错，还得记住坑人的新建文章命令什么的，下载markdown，开始写markdown，算是和之前的typecho比起来，比较繁忙的一点了吧。但是免费这一点也不错，各有各的好处把。
